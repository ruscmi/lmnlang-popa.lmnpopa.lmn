Первым делом я разберу вам простой код,выводящий "Hello,world!" на языке C++

C++ main.cpp
------------------------------
#include <iostream>

int main() {
    std::cout<<"Hello,world!";
    return 0;
}
------------------------------
Если вы ничего не поняли,это нормально,первым делом обьясню что за "#include <iostream>" - это базовая библиотека,которая заставляет 
код быть рабочим,иными словами она "открывает вход-выход" между тем что вы пишите в main() и С++.
int main() - создание базовой функции,которая в свою очередь отвечает за все что вы пишите в коде.
std - пространство для имен(cout,cin,vector,map...).
cout - вывод слов.
return 0 - прекращение работы программы(возвращение к нулю). 
{} - скобки использующиеся для открытия и закрытия основной функции main().
<< - эти "стрелочки" говорят направь в cout текст в кавычках.
:: - "4 точки" обозначают использования префикса для std(в данном случае cout).

Попробуйте записать этот код в текстовый файлик,названный main.cpp,а потом скомпилировать и запустить его.

g++ main.cpp -o main

./main

Если у вас вывело "Hello,world!" поздравляю! Это ваш первый рабочий код в данном инструменте от бога.

Рассмотрим интересный способ сокращения синтаксиса кода.
C++ main.cpp
---------------------
#include <iostream>
using namespace std;
int main() {
    cout<<"Hello,world!";
    return 0;
}
--------------------

Что произошло с кодом? 
Я добавил в начало,до создания основной функции main() правило для кода: "using namespace std".
Что оно делает? 
Вы скорее всего могли заметить что пропал std. Это произошло потому что using namespace std прямо говорит:
 "Используй пространство имен std"
Это позволяет нам значительно сокращать код,но иногда это может сильно мешать вам его читать.

                                             СОЗДАНИЕ ПЕРЕМЕННЫХ
Что такое переменная?
Переменная это как пустая коробочка,в которую можно положить какую то вещь.
Переменную можно задавать и как числа(int) и как строку со словами (string)
Вот простой код с работой переменных.

С++ main.cpp
----------------------
#include <iostream>
#include <string>
int main() {
    int year = 10;
    std::string name = "Denis"; 
    std::cout<<"Привет,мне "<<year<<" лет,меня зовут "<<name<<std::endl;
    return 0;
}
-----------------------

Что произошло в коде?
Первым делом мы импортировали библиотеку для работы с строчками <string>.
Мы записали в переменную year значение 10 и в переменную name имя Denis.
Я так же добавил std::endl,это позволит коду переносить вас на следующую строчку.
Переменные это очень важная часть работы с кодом,она вам будет нужна в более серьезных проектах,но об этом позже...
Запустите код так же,как я указал ранее,если вам выдало надпись "Привет,мне 10 лет,меня зовут Denis" - код работает.

                                                      ГЛАВА 2.
                                                 Нырок в углубление.
В этой главе я бы хотел поговорить о разных интересных возможностях С++ как языка.

Пожалуй надо начать с того что сделает ваш код более интересным,но одновременно не сломает ваш мозг :)

В многих языках есть возможность записи значения в переменную прям при работе кода,таким процессом часто назначают input(),но вот в C++ это работает немного
иначе с помощью cin.

Что такое cin?
cin - как обратный cout,вместо того что бы выводить текст,мы его записываем.
Вот вам простой пример кода с использованием записи в переменную 

С++ main.cpp
-----------------------------
#include <iostream>
#include <string>
int main() {
    int year_old;
    std::cout<<"Сколько тебе лет?: "<<std::endl;
    std::cin>>year_old;
    std::cout<<"Тебе "<<year_old<<" лет"<<std::endl;    
    return 0;
}
-----------------------------

Данная программа спросит ваш возраст и выведет его в таком формате: "Тебе ваш_возраст лет"
Это позволяет делать различные проверки авторизации и многое другое.
Что вообще произошло в коде?
Я создал пустую переменную year_old,куда впоследствии можно записывать числа,в данном случае это ваш возраст.
Почему именно >>?
>> - это как у cout,но наоборот,тоесть мы буквально говорим С++ не выводить текст,а записывать его в переменную year_old.

Это достаточно интересно,не правда ли?
Если вам на этом месте стало не понятно что либо,это нормально,С++ - низкоуровневый язык,да он сложный,но честный.
Придумаем ситуацию когда вам надо спросить у друга как его зовут и какой у него возраст,в С++ это можно записать следующим образом:

C++ main.cpp
-------------------------
#include <iostream>
#include <string>
using namespace std;
int main() {
    int year_old;
    string name;
    cout<<"Как тебя зовут?: ";
    cin>>name;
    cout<<"Сколько тебе лет?: ";
    cin>>year_old;
    cout<<"Привет, "<<name<<" тебе "<<year_old<<" лет "<<endl;
    return 0;
} 
--------------------------

Кстати? Если вы начали задаваться вопросом в чем же прелесть этого сложного динозавра из 80-х годов,который используется везде
то я вам отвечу прямо так:
С++ - низкоуровневный язык,он позволяет коду напрямую работать с железом,из за этого С++ используется там,где это жизненно необходимо.
Представьте если бы вы написали ОС на Python для работы с дисплеем автомобиля...
Да уж,такое страшно представить,ведь в отличии от С++,Python является интерпретатором,он работает на высоком уровне,именно из за этого он бы
не смог стать конкурентом С++ в сложнейших задачах,где важна скорость.
Ладно,переходим к следующей тематике.

                                                      КОММЕНТАРИИ В КОДЕ
Скорее всего,если вы знакомы с такими языками как Python,Ruby,Lua,JavaScript и другими,содержащими в себе комментарии вы бы подсчитали данную часть 
главы бесполезной, но те кто понимают значение комментариев в коде оценят мое объяснение.
Комментарии в кодах обычно используются для объяснения исходного кода,который могут не понять новички,его так же используют что бы временно "глушить"
строчки кода,которые вам не нужны.
Обычно комментарии используют как то вот так:
C++ main.cpp
--------------------------
#include <iostream> // Библиотека для ввода - вывода
using namespace std; // Говорим: используй наименование std для каждого имени
int main() { // Основная функция
	cout<<"Привет,я Денис!"<<endl; // Выводит предложение
    return 0; // Заглушаем работу кода
} 
// По желанию можно вмещать комментарии даже сюда
// Или сюда,их просто будет игнорировать код.
--------------------------
В С++ комментарии обозначаются с двумя слешами,тоесть вот таким образом:
// Ваш комментарий
В общем,комментарии лучше использовать там,где вы обычно запинываетесь и забываете логику.
Я использую комментарии для более крупных проектов,где они необходимы.

                                            ТИПЫ ДАННЫХ
На самом деле,типов данных в С++ достаточно не мало,но я перечислю основные,которые лучше всего использовать в коде,если вам важна скорость,прямая работа с памятью и легкость кода.
Каждый тип данных имеет какой либо размер,зачастую несколько бит.
int - целое число (размер около 4-х байт)
float - неправильная дробь(размер около 4-х байт)
double - такая же неправильная дробь,позже объясню различия(размер около 8-ми байт)
string - строка(разный размер)
char - очень маленький тип данных,хранящий всё по 1 символу (весит 1 байт)
Type[//число] - массив
vector - массив с бесконечным донышком
Скорее всего вам не понятно что за массив? 
Массив - коробка содержащая в себе несколько предметов,обычный массив ограничен тем,что вы напишите в коде,в отличии от vector,являющийся динамическим и бесконечным
в содержании текста. О массивах вы узнаете позже,а пока я должен расказать про остальные типы данных.
int,float,double используются в математических действиях и вычислениях. 
Они нужны везде,банкоматы,финансовые дела и другие подобные дела.
char - малюсенькая переменная,в которую ты можешь впихнуть только 1 символ(если это не массив).
string - это строчка,я ранее показывал работу со string,мы можем записывать в нее различные слова и символы,даже числа можно(но потом их приходится переводить через stoi)
В общем типы данных - очень важная тематика.
Вы позже столкнётесь с ними,но пока что это все,что вам нужно знать.

                                                     ОПЕРАТОРЫ ВЫРАЖЕНИЙ
Что вообще такое операторы выражений? Какой то страшный термин... На самом деле,операторы выражения это то,что необходимо понимать в сравнении,присваивании,
приравнивании и других действиях с логическими выражениями. Я покажу лишь основные примеры,которые вам точно понадобятся.
Какие бывают операторы выражений?
Арифметические - используются в математических вычислениях(в том числе в калькуляторе)
Вот основные операторы:
 "+" - сложение,все как в алгебре,мы просто складываем числа или переменные.
 "-" - вычитание,работает как в алгебре.
 "*" - умножение,та же самая работа.
 "/" - деление,те же действия.
 Данные операторы используются в калькуляторах,вычислениях возраста при регистрации пользователей и в других различных интересных работах кода.
Присваивание и сравнение:
 "==" - используется для сравнения двух переменных,массивов и в прочих типах данных.
 "=" - присваивание,главное не путать со сравнением.
 "!=" - обозначает не равенство.
 ">=" - больше или равно.
 "<=" - меньше или равно.
 ">" - больше.
 "<" - меньше.
 Данные операторы используются в сравнениях чего либо,в назначениях переменных и в многом прочем.
Логические:
 "&&" - обозначает И,тоесть например Петя&&Денис - здесь && является значением И.
 "||" - обозначает Или,тоесть Петя||Денис - здесь || является значением Или.
 "!" - восклицательный знак обозначает отрицание(NOT).
 Данные операторы используются в разных случаях,которых очень много,они помогают нам передавать переменные совместно,или ставить отрицание в чем либо.

                                                              ГЛАВА 3
                                                         Углубление в дыру
Ох... Данная глава должна получиться очень крупной и массивной по сравнению с другими,тут я объясню как создавать проверки 
делать свои авторизации,вводы паролей и даже калькулятор.

Хорошо,представьте себе ситуацию:
         Вы говорите бабушке что к вам придет друг,например по имени Арсений,вы говорите:
         - Если тот кто к тебе придёт назовёт себя Арсений - это он.
         - А если иначе,то это не он.
Так же и в коде можно взаимодействовать с if и else.
Что такое if и else?

if - обозначает значение "если".
else - обозначает значение "иначе".

Представим ситуацию что вам нужно сделать такую проверку на С++.
Это выглядело бы вот так:
C++ main.cpp
--------------------------------
#include <iostream>
#include <string>
using namespace std;
int main() {
    string name;
    cout<<"Как тебя зовут?: ";
    cin>>name;
    if(name == "Denis") {
    	cout<<"Это он!"
    }else {
    	cout<<"Это не он..."
    }
    return 0;
}
--------------------------------
Вот такая проверка на имя Denis,если пользователь введет Denis ему выдаст "Это он!".
Если же нет то выведется предложение "Это не он..."
Я думаю что вы к этому моменту должны были понять как работает if else,а если вы до этого изучали
что-то похожее на С++,например Python,то вы давно бы это поняли.
Если же нет,предлагаю представить еще одну ситуацию:
             Вы приходите к другу и говорите ему:
             - Привет,наш кодовый номер от базы 1234,скажи его всем нашим друзьям.
             - Если кто-то ответит неправильно то просто выгони их из нашей базы.
             - Если кто то ответит верно,просто впусти их и скажи "Всё успешно!".
Представим это в С++...
Нам надо сделать код который будет проверять наш кодовый номер,
если все правильно,то он нас пропустит,если нет,то он сбросит программу и выгонит нас.
Вот как бы это выглядело на С++:
C++ main.cpp
-----------------------------------------
#include <iostream>
#include <string>
int main() {
	std::string password;
	std::cout<<"Введите пароль: ";
	std::cin>>password;
	if(password == "1234") {
		std::cout<<"Всё успешно!";
	}else {
		std::cout<<"Проваливай отсюда! ";
        return 0;
	}
}
Так,теперь разберем что такое break? break - это если бы мы программе сказали: выйди из текущего цикла.
Если вы хотите дополнить код повыше новыми проверками,вам необходим break внутри else.
Так слушайте,я думаю нам стоит заканчивать с данной тематикой,но мы скоро к ней вернемся.
Переходим к циклам,это очень интересная тема,которая позволяет вам не выходить из программы и использовать ее либо бесконечно,
либо сколько раз вы захотите.

                                                                   ЦИКЛЫ

Представьте жизненную ситуацию вам надо сказать другу: прыгни 5 раз.
Вы будете считать его количество прыжков,пока оно не стукнет 5-ти.
Если оно стукнуло 5-ти,он выполнил твоё поручение.
А теперь вот как бы это выглядело в С++:
C++ main.cpp
---------------------------------
#include <iostream>
#include <string>
int main() {
	int jump = 0;
	while(jump < 5){
		jump++;
		std::cout<<"Прыжок!"<<std::endl;
	}
	std::cout<<"Всё,ты допрыгал!";
	return 0;
}
---------------------------------
Запустите данный код на вашем компьютере,вы увидите примерно такой результат:

Прыжок!
Прыжок!
Прыжок!
Прыжок!
Прыжок!
Всё,ты допрыгал!

Теперь перейдем к сути, что такое вообще while?
while - цикл,который имеет какое либо условие и используется зачастую для создания бесконечного пространства 
в программе и выхода из неё.
В коде мы говорим: пока значение меньше 5 - Прыгай! 
while(jump < 5) - прямым контекстом нам говорит,запусти цикл,пока значение не будет равно 5.
jump++ - Увеличивает значение,что делает цикл не бесконечным,а пока действует условие.
На самом деле это очень нужная тематика,так как она используется везде.
Но while - не единственный цикл.
У него есть еще и брат,о котором я сейчас вам расскажy...

Представьте себе ситуацию:
Вам в коде надо вывести 100 слов "Привет!",вроде как вы можете это сделать и с помощью
std::cout<<"Привет"; 
А вроде бы,вы понимаете,что это очень долго,займет огромный размер и сожрет ваш синтаксис кода.
Для таких случаев придумали цикл for,это цикл который позволяет тебе выводить очень много слов,
выводить массивы или переменные как ты хочешь и проходиться по всему массиву.

Логика такая:

Получаем знак операции

На С++ такая логика была бы не сильно простой,по сравнению с тем же Python,но и не сложной.

С++ calculator.cpp
--------------------------------------
#include <iostream>
#include <string>
using namespace std;
int main() {
    double number_one;
    double number_two;
    string operation;
    string exit;
	while(true) {
		cout<<"Хотите ли вы выйти?(Д/Н): ";
		cin>>exit;
		if(exit == "Д") {
			break;
		}else {
			cout<<"Погнали!"<<endl;
		}
		cout<<"Введите первое число: ";
		cin>>number_one;
		cout<<"Введите второе число: ";
		cin>>number_two;
		cout<<"Введите знак операции: ";
		cin>>operation;
		if(operation == "+") {
			cout<<number_one + number_two<<endl;
		}
		else if(operation == "-") {
			cout<<number_one - number_two<<endl;
		}
		else if(operation == "*") {
			cout<<number_one * number_two<<endl;
		}
		else if(operation == "/") {
		    if(number_two != 0) {
			    cout<<number_one / number_two<<endl;
            }
		}
	}
	return 0;
}
-----------------------------------
Да уж,код выглядит так будто вы это никогда не напишите,но на самом деле,там все гораздо проще,чем вы думаете.
В начале кода мы создаем переменные,куда в последствии мы записываем значения,в этом случае 2 числа и операция.
потом мы проводим проверку на выход,если вы отвечаете "Д" цикл заканчивает работу,а потом и сам код. 
Если вы отвечаете "Н" или что то другое он просто дружелюбно продолжает работать и далее мы подходим к самой логике.
Она здесь примерно как на пайтоне: мы сравниваем операцию с арифмитическими операциями.
Кстати я использую double,он более точный и необходим для огромных чисел,в отличии от int с ограниченным размером. 
Так ну тут все более понятно: если пользователь пишет в переменную operation знак "+" - мы складываем number_one и
number_two,если мы пишем "-",то производится операция вычитания. Все работает как часы,я даже добавил проверку деления на 0(Это критично!)
Но знаете что,хоть мы пока что это и не проходили,но в будущем мы добавим в него еще и try && catch,это такие дебагеры для кода,проверяющие не вводим
ли мы что то не то,потому что если мы сейчас введем в переменную number_one не число,а например имя Денис,то нам выдаст ошибку сегментации.

Что такое функция?
Представьте себе ящик в котором будут лежать ваши рабочие вещи.

                                                                              MAP
Что такое мапа?
Представь обычный бумажный словарь. Ты ищешь слово (это Ключ) и видишь его перевод или описание (это Значение).
В мапе всё хранится парами: Ключ -> Значение.

Мапа -  уже не просто какой нибудь массив. Это твоя личная база данных.
Проще говоря,мапа это как словарь в Python.
Вот пример использования мапы в С++

С++ main.cpp
--------------------------------------------
#include <iostream>
#include <string>
#include <map>
using namespace std;
int main() {
    map<string,int> fruits;
    fruits["lemon"] = 3;
    fruits["apples"] = 10;
    fruits["banans"] = 5;
    for(auto const& pait : fruits) {
    	cout<<pait.first<<" : "<<pait.second<<endl;
    }
    return 0;
}
---------------------------------------------
Мы буквально создаем нашу память для фруктов,помещаем туда фрукты со своим значением,а после чего выводим
и фрукты и их количество.
Это достаточно удобно,но может быть замудренно для новичков.

                                                               УКАЗАТЕЛИ В ПАМЯТИ

Что такое вообще эти ваши указатели?
Представьте что что ваша оперативная память содержит в себе адреса различных домов и 
их номера квартир.

Тоже самое и с указателями. 
Когда ты создаёшь переменную например int x = 42;, компьютер:
    Находит свободную ячейку (например, с адресом 1000)
    Кладёт туда число 42
    Запоминает, что имя x связано с адресом 1000

Зачем нужны операторы и что это?
Операторы позволяют нам управлять ячейками в памяти и смотреть на их адреса и значения.

& - этот оператор говорит коду "Дай мне адрес".
* - этот же говорит "Пройдись по адресу".

Пример работы кода с оператором "&"

C++ main.cpp
------------------------------------------------
//это всего лишь отрывок
int x = 14;
cout << &x; //выведет номер ячейки памяти
------------------------------------------------

Пример работы кода с оператором "*" 

С++ main.cpp
-----------------------------------------------
// отрывок
int x = 14;
int* ptr = &x;
cout << &ptr;
-----------------------------------------------

*ptr читается как "значение по адресу ptr".

Схема памяти:

Переменная x:   [адрес: 1000] → значение 5
Указатель ptr: [адрес: 2000] → значение 1000 (адрес x)

Вот мини словарь на запоминание

Символ	Название	Что делает
&x	    "амперсанд"	взять адрес переменной x
*ptr	"звездочка"	взять значение по адресу ptr
int&	"ссылка"	псевдоним переменной (не копия)
int*	"указатель"	переменная, хранящая адрес

Как вообще работает интерпретатор?
Первым делом мы создаем лексер.
Что такое лексер?
Лексер - часть интерпретатора разбивающая команду на кусочки с помощью прохождения по нему.
Зачем нам нужен лексер?
Что бы в памяти хранить значения для переменных.
Первым делом нам надо создать лексер,после чего переходить к парсеру.
Вот что у нас должно получиться

C++ lexer.cpp
----------------------------------------
//отрывок кода из исходного кода lmnlang
stringstream ss(inpline);
string w;
while(ss>>w) {
	v.push_back(w);
}
----------------------------------------

Вообще это просто часть,но она работает и делает работу лексера.
Что происходит в коде?
Мы создаем stringstream в который мы помещаем inpline на разделение.
inpline - строка которую мы разбиваем.
Логика работает таким образом:
   - Наш лексер получает "set x 10"
   - Разбивает его на части
   - Помещает в конец вектора

Вот мы уже создали рабочий лексер,который вы потом будете использовать в своем коде.
Далее мы переходим к созданию парсера.
Что вообще такое парсер и зачем он нужен?
Парсер - тот кто понимает смысл наших команд.
Тоесть он буквально видит строчку "set x 10" и переводит ее для себя как:
"Сохрани в память 10".
Парсер это по сути наша цепочка для проверки команд,я приведу пример самой простой команды которая создает
переменные и запоминает их в память.
Вот пример "set" из lmnlang

C++ parser.cpp
----------------------------------------
if(v[0] == "set") {
	m[v[1]] = v[2];
	cout<<v1<<" = "<<m[v[1]]
}
----------------------------------------
Смотрите как это работает
Допустим пользователь вводит "set x 10"
Лексер разбивает строку на вектора
v[0] - наш set
v[1] - ключ (например "x")
v[2] - значение(например 10)
Таким образом мы получаем связь двух работяг,которые помогают нам выстраивать
примерно такую логику работы интерпретатора:

"set x 10"  (лексер) ->  ["set", "x", "10"]  (парсер) ->  сохранить x = 10

А что вообще за m[v[1]] = v[2]; и v[1] ?
m[v[1]] - ячейка в нашем словаре(мапе) по ключу(например "х")
v[2] - третье слово команды(значение)
v[1] - второе слово команды(ключ)

Эта казалось бы небольшая логика позволяет нам запоминать в память много аргументов и одновременно не портить
синтаксис,производя такую работу.

Начало основной функции
int main() {
    bool starting = true;  // флаг: пока true — программа работает
    std::map<std::string, std::string> memory;  // словарь для переменных
    while (starting) {   // пока `starting` не станет false
        std::vector<std::string> variables;  // вектор для токенов
        std::string inpline;                  // строка от пользователя
        std::cout << "input('exit' for stop): ";
        std::getline(std::cin, inpline);     // читаем всю строку целиком
        // === ЛЕКСЕР: разбиваем строку на токены ===
        std::stringstream ss(inpline);   // загружаем строку в поток
        std::string words;               // временная переменная для каждого слова
        
        while (ss >> words) {            // пока есть слова — забираем
            variables.push_back(words);  // кладём в вектор
        }
        if (variables.empty()) continue;  // если ничего не ввели — просто ждём дальше
        // === ПАРСЕР: понимаем команды ===
        if (variables[0] == "exit") {
            starting = false;   // выключаем цикл
        }
        else if (variables[0] == "set") {
            if (variables.size() == 3) {  // должно быть 3 слова: set, имя, значение
                memory[variables[1]] = variables[2];
                std::cout << variables[1] << " = " << memory[variables[1]] << std::endl;
            } else {
                std::cout << "ValueError" << std::endl;
            }
        }
        else if (variables[0] == "print") {
            for (size_t i = 1; i < variables.size(); i++) {
                if (memory.count(variables[i]) > 0) {
                    std::cout << memory[variables[i]] << " ";
                } else {
                    std::cout << variables[i] << " ";
                }
            }
            std::cout << std::endl;
        }
    }
    
    return 0;
}
